# Flue Framework source excerpts for eval reproducibility

User prompt under test:

> I want to create a poster for the Flue Framework:
> https://flueframework.com/
> https://github.com/withastro/flue

Verified source facts from the public site/repository at the time this eval was added:

- Product/title: Flue.
- Positioning line: "The Agent Harness Framework."
- Primary claim: "Not another SDK. Build autonomous agents and powerful AI workflows with Flue's programmable TypeScript harness."
- Flue is described as a TypeScript harness for autonomous work with sessions, tools, skills, instructions, filesystem access, and a secure sandbox.
- The site presents a Universal Agent Architecture with four layers:
  1. Model — tokens, tools, prompts.
  2. Harness — skills, memory, sessions.
  3. Sandbox — bash, security, network.
  4. Filesystem — read, write, grep, glob.
- Feature copy says agents can keep context across conversations and events while autonomously working toward a goal.
- README package list includes:
  - `@flue/runtime` — runtime: harness, sessions, tools, sandbox.
  - `@flue/cli` — CLI and build/dev tooling.
  - `@flue/sdk` — client SDK for consuming deployed agents and workflows.
  - `@flue/postgres` — Postgres persistence adapter.
- README deploy targets include Node.js, Cloudflare Workers, GitHub Actions, GitLab CI/CD, Daytona, and Render.
- GitHub repository metadata: `withastro/flue`, language TypeScript, license Apache-2.0.

Web-CTA caution:

- The website contains web-navigation/action language. A poster should not blindly turn website UI buttons such as "View Documentation" into the poster's call to action. If a CTA is needed, it should be a poster-appropriate instruction derived from the source facts, or the source facts should be presented without a CTA button module.
