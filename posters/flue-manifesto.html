For a generation we wrote against the model: a script of calls, a chain of prompts, a brittle pipeline that did exactly what we said and never more. That is a software development kit. It is not an agent. An agent is something we hand a task rather than a procedure — and then trust to find its own way.

The distinction is simple and absolute. A model, on its own, only reasons; it thinks across the problem and proposes the next action. To turn that proposal into work it needs a place to act, and the means to act there. The harness is that place and those means. It frames the model with context, hands it typed tools, loads the skills it needs, and opens a secure sandbox where shell and filesystem are real.

Around this it builds what no single call can hold: sessions and memory that persist across turns and across events, so the work survives between one moment and the next. The agent does not begin again each time it is woken. It returns to what it knew.

And it is given a loop to run on. The model reasons over the task; the harness lets it act; the result is observed and folded back into the reasoning — reason, act, observe, and again, until the goal is met. This is the whole machine. The model is the mind; the harness is everything the mind needs in order to be of any use at all.

So we give it a goal and not a script. It acts. It remembers what it has done. When something fails it does not halt — it observes, adjusts, and recovers, working autonomously toward the end we named. That is why this is not another SDK. It is the harness an agent runs inside.

You give it a task, not a script. It acts, remembers, and recovers — working on its own toward a goal.
