MEDIA OVER QUIC

A new path through the latency-versus-scale trade-off. Sub-second live media fan-out: WebRTC-like responsiveness, CDN-style relay scale, one media-agnostic object model.

MoQ is not a codec, player, CDN, DRM system, or ad stack — it is a media-agnostic transport / object model.
01The old trade-off
Positioning is directional. Actual latency and fan-out depend on implementation, network, player, and relay deployment.
Sub-second target.
CDN-style fan-out.
Incremental deployment.
Works alongside today's stack; replaces custom real-time glue over time. Fewer trade-offs.
02Data model
Track
A subscribable named sequence of groups — e.g. video/1080p, audio/en, metadata/scores.
Group
A join / cache boundary within a track; for video, often GOP-like or independently useful.
Subgroup
Ordered objects in a group with shared dependency / priority, usually carried on one stream.
Object
An immutable addressable payload plus relay-visible metadata.
Relays forward objects without decoding media.
Payload may be encrypted; object metadata remains relay-visible.

05Adopt incrementally
  1. Keep current ingest: RTMP / SRT / WHIP.
  2. Add MoQ egress for modern browsers and apps.
  3. Keep HLS / DASH fallback for legacy devices.
  4. Measure latency, rebuffering, cost per viewer-hour, engagement.
06Not solved by transport alone
DRM · SSAI / DAI · captions · QoE analytics · auth · compliance · multi-CDN routing. Transport is the floor.
