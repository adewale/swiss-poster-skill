The Claim00

One object = one atom of coordination.

Name it once; reach it from anywhere. It owns its slice of state, runs single-threaded, and answers to nothing else — ~500–1,000 req/s per atom, by design.

01 / The Collapse/01

Six services become one object.

The classic stateful backend is a federation of moving parts — each with its own consistency model, failure mode, and bill. A Durable Object replaces every row below with a single addressable runtime.

Redis  — in-memory cache & locks→Object memory
Postgres row  — per-tenant state→Object storage (SQLite)
RabbitMQ / SQS  — queue per actor→Single-threaded inbox
WebSocket fleet  — sticky routing→Hibernating connections
Cron / scheduler  — external trigger→setAlarm()
ZooKeeper  — distributed locking→Serial execution

04 / Why Atoms Win/04

The case for atoms.

Strong by default, not by ceremony.

One writer, one reader, one queue. The hardest distributed systems problem becomes a single-threaded function. You write code; the platform stops lying to you about ordering.

State follows compute. Compute follows users.

The object boots near the first request and stays. Reads are local. Writes are local. The 50 ms cross-region round-trip you budgeted for is now a function call.

Idle is free. Active is cheap.

No instance to provision, no fleet to autoscale, no pre-paid Redis. The object exists when invoked, hibernates when not, and pays per request & per GB-second of state.

export class Counter extends DurableObject {
  constructor(ctx, env) {
    super(ctx, env);
    this.sql = ctx.storage.sql;
    this.sql.exec(`CREATE TABLE IF NOT EXISTS
      n (v INTEGER NOT NULL)`);
    this.sql.exec(`INSERT OR IGNORE
      INTO n VALUES (0)`);
  }

  // One CPU. One disk. One name.
  async increment(by = 1) {
    return this.sql.exec(
      `UPDATE n SET v = v + ?
         RETURNING v`, by).one().v;
  }
}

// From anywhere on Earth:
const id   = ns.idFromName("counter");
const stub = ns.get(id);
await stub.increment();  // strongly
await stub.increment();  // consistent.
