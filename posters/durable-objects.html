02 Anatomy / inside one object
any client → idFromName("standup") ↓
/room/standup the named instance · born near its first request
storage
SQLite on its own disk, co-located. Storage calls are function calls — no network in the read path.
memory
Class fields. Last until hibernation; re-hydrate from storage on wake.
inbox
Requests handled one at a time, in arrival order. No locks, no races — single-threaded authority.
alarms
Scheduled work that survives sleep. ctx.storage.setAlarm().
sockets
WebSockets that hibernate. Idle connections cost nothing; the next message wakes the object.
▽  asleep is the default state · wakes on the next request · costs nothing while idle  ▽

04 What follows / three consequencesscale · sleep · speed

Scale is more names.

You don't grow one database — you mint more objects. A million rooms is a million objects, each small, each owning its slice.

Consistency is free.

One object, one CPU, one inbox. Within a thing, there is nothing to coordinate — ordering comes from the architecture, not from locks.

Idle is nothing.

Sleep is the default state, not an optimization. An object exists while it's needed and costs nothing while it isn't.

A Durable Object is what you get when a class has a name and its own disk
