Three tiers
Virtual · Local · Remote
low → high isolation
01
Virtual
sandbox: virtual()
lightweight · in-memory · just-bash
An ephemeral shell with no host access — the default. Nothing the agent runs touches a real disk or machine.
use when
Most agents. Scripted steps that never need a real toolchain.
02
Local
sandbox: local()
host filesystem + shell · direct
Runs against your machine: real files, real commands, full speed. The trust you grant is the host itself.
use when
Trusted dev & CI on code you own and control.
03
Remote
sandbox: remote()
provider-managed isolated container
A fully isolated Linux box (Daytona, Cloudflare Sandbox) — a real toolchain walled off from everything you own.
use when
Untrusted work that still needs a genuine environment.
The guiding rule
Grant only the reach the work demands — climb one step, never two.
