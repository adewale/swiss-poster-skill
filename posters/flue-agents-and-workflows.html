Agents & Workflows.

They aren't opposites — they invert control. A Workflow is code that calls a model; an Agent is a model that calls your code. Both run the same harness (loop: Pi), so you dial autonomy per task — script what you know, hand the rest to the model, and wrap one inside the other.
A sub-agent is an agent nested in another's subagents[] — same shape, but no durability of its own and checked for cycles. The parent calls a declared one by spawning a task session; depth is bounded (TaskDepthExceededError).
What Pi contributes the engine
Flue's loop is Pi — the agent harness behind OpenClaw, packaged as @mariozechner/pi-agent-core. Pi runs the reason → act → observe cycle and the tool calls that let an agent act on its own. Flue keeps that engine and builds the rest around it: Durable Streams for state, a Sandbox API, channels, and the Agents/Workflows model — so the same agent deploys to any runtime.
What stays the same the harness, invariant
One createAgent() defines the agent both use. Model, tools, skills, instructions, and sandbox are identical whether the model drives it or your code does.

No run/defineWorkflow export: a Workflow is a file whose run(ctx) the runtime invokes. init is a method on that FlueContext — init(agent) → FlueHarness — so a Workflow boots an Agent, then session().prompt().
Caveat: AgentProfileSchema types model?: string | false — an agent can be configured without a model. The abstraction is model-driven; the requirement isn't absolute.
Why it mattersControl is a dial, not a fork.
Autonomy per task
Keep the parts you know deterministic — a Workflow returns a typed result with a replayable runId. Give the open-ended parts to an Agent. One project, not two frameworks.
Write the harness once
The same tools, skills, sandbox, and model back both. Promoting a step from scripted to autonomous — or pinning it back down — needs no rewrite.
Guardrails around autonomy
A Workflow drives an Agent for the fuzzy step, then resumes as typed, recoverable code — a predictable shell around an unpredictable core. That's what ships to production.
