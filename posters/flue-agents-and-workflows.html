The agent is the unit · the workflow runs it

Define an agent once.
Drive it two ways.

An Agent is the unit — a model wrapped in a harness (tools, skills, a sandbox). A Workflow is plain code that calls agents and sequences them to a result. Serve the agent and the model drives it; call it from a workflow and your code drives it. Same agent, same harness.

export default createAgent(() => ({

src/workflows/triage.ts — code that runs it
import triageAgent from '../agents/triage';

export async function run({ init, payload }) {
  const s = await (await init(triageAgent)).session();
  return (await s.prompt(payload, { result })).data;
}
An agent is the unit. A workflow runs agents.
Write the agent once. Serve it and the model drives its loop, keeping a session. Call it from a workflow and your code drives — prompting it, sequencing several, returning a typed result. An agent can also delegate to sub-agents. Underneath, every path is the same harness: init(agent) → session() → prompt().
